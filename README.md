# Gmailメールマガジン整形転送システム

Gmailで受信したテキスト形式のメールマガジンをHTML形式に変換し、Blogger（ブログサービス）へ転送するためのGoogle Apps Script (GAS) です。

## 機能概要

- **自動HTML変換**: テキスト形式のメールマガジンの改行を `<br>` タグに変換し、Blogger経由のRSSリーダーで改行が消える問題を解決します。
- **URLリンク化**: 本文中のURLを自動的に検出し、クリック可能なリンク (`<a>` タグ) に変換します。
- **既読管理**: 指定したラベルが付与された「未読」メールのみを処理し、処理後は「既読」にすることで重複処理を防止します。

## セットアップ方法

### 1. スクリプトの作成
1. [Google Apps Script](https://script.google.com/) にアクセスし、新しいプロジェクトを作成します。
2. `Code.gs` の内容をプロジェクトに貼り付けて保存します。

### 2. スクリプトプロパティの設定
スクリプトの「プロジェクトの設定」（歯車アイコン）から「スクリプトプロパティ」に以下の2つを追加します。

| プロパティ名 | 設定値の例 | 説明 |
| :--- | :--- | :--- |
| `BLOGGER_ADDRESS` | `example.user.secret@blogger.com` | Bloggerの投稿用メールアドレス |
| `DRY_RUN` | `true` | `true`に設定すると、転送や既読化を行わずに動作ログのみ出力します |

### 3. メルマガ転送設定の編集
`Code.gs` 内の `MAIL_MAGS_CONFIG` 定数を編集して、処理対象のラベルや実行曜日を設定します。

```javascript
const MAIL_MAGS_CONFIG = [
  { label: "mailmag", days: [], hours: [], convertHtml: true },
  { label: "mailmag-NikkeiBP", days: [1], hours: [11], convertHtml: false },
  // ...
];
```

### 4. トリガーの設定
定期的にメールをチェックするために、トリガーを設定します。
1. スクリプトエディタの左メニューから「トリガー」（時計アイコン）を選択します。
2. 「トリガーを追加」をクリックします。
3. 以下の設定で保存します。
   - 実行する関数: `main`
   - デプロイ: `Head`
   - イベントのソース: `時間主導型`
   - トリガーのタイプ: `分ベースのタイマー`（例: 15分おきなど）

## 使い方
1. 転送したいメールマガジンに対して、`MAIL_MAGS_CONFIG` で指定したラベルが付与されるようにGmailのフィルタを設定します。
2. スクリプトは、設定された「実行曜日」かつ「実行時間」にのみ、対象ラベルの「未読」メールを処理し、処理後に「既読」にします。
3. 設定したトリガーに従って、自動的にメールが変換・転送されます。

### 親ラベルと子ラベルの扱い
ラベル名が他のラベルのプレフィックスになっている場合（例: `mailmag` と `mailmag/nikkepbp`）、より具体的なラベル（子ラベル）が付与されているメールは、親ラベルの処理対象から自動的に除外されます。これにより、意図しない二重処理を防ぎます。
- `mailmag/` または `mailmag-` で始まるラベルが付いている場合、`mailmag` の設定では処理されません。

## 注意事項
- 初回実行時には、Gmailへのアクセス権限などの承認を求められる場合があります。
- 大量のメールを一度に処理しようとすると、GASの実行時間制限（通常6分）に抵触する可能性があります。本スクリプトでは1回あたり1ラベルにつき最大1件に制限しています。
- 7日以上前の古いメールは処理対象から除外されます。
